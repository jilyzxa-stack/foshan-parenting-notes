## 🎞 动画标准（三段式结构，每页必须完整实现）

| 阶段 | 效果 | 时长 |
|---|---|---|
| **入场** | 从下方滑入 + 透明度渐显（`translateY` + `opacity`） | 15 帧（0.5s） |
| **展示** | 静态停留，或逐行 / 逐项依次出现（stagger） | 根据内容量决定 |
| **出场** | 向上滑出 + 透明度渐隐 | 15 帧（0.5s） |

### 标准动画代码模板

```jsx
import { useCurrentFrame, interpolate, Easing } from 'remotion';

// 入场动画（帧 0–15）
const enterProgress = interpolate(frame, [0, 15], [0, 1], {
  easing: Easing.out(Easing.cubic),
  extrapolateLeft: 'clamp',
  extrapolateRight: 'clamp',
});

// 出场动画（最后 15 帧）
const exitProgress = interpolate(frame, [totalFrames - 15, totalFrames], [0, 1], {
  easing: Easing.in(Easing.cubic),
  extrapolateLeft: 'clamp',
  extrapolateRight: 'clamp',
});

const translateY = enterProgress === 1
  ? interpolate(exitProgress, [0, 1], [0, -60])
  : interpolate(enterProgress, [0, 1], [60, 0]);

const opacity = enterProgress < 1
  ? enterProgress
  : interpolate(exitProgress, [0, 1], [1, 0]);

// stagger 逐项出现（展示阶段，每项延迟 8 帧）
const itemDelay = index * 8;
const itemProgress = interpolate(frame, [15 + itemDelay, 30 + itemDelay], [0, 1], {
  easing: Easing.out(Easing.back(1.5)),
  extrapolateLeft: 'clamp',
  extrapolateRight: 'clamp',
});
```

## 🔄 工作流程（必须严格遵守，不得跳过）

### 第一步：脚本分析（收到脚本后立即执行）

仔细阅读全文，识别并提取：

1. **主要阶段 / 章节**：视频分几个部分，各自核心主题是什么？
2. **每个阶段的关键点**：最值得视觉化放大的 1–3 个信息点
3. **适合视觉页的内容**：数据、列表、步骤、对比、金句
4. **适合图表的内容**：有具体数字、百分比、趋势的内容
5. **应该删减的内容**：过渡性语言、口语填充词、重复表述、无信息量的过渡句

## ✅ 代码质量要求

1. **每个组件必须接收 `totalFrames` prop**，用于计算出场动画时机
2. **所有文字必须有明确的对比色保证可读性**，禁止低对比度文字
3. **禁止硬编码颜色**，所有色值定义为顶部常量：
   ```jsx
   const BRAND = {
     bg: '#ebbe67',
     accent: '#1c78fd',
     dark: '#1a1a2e',
     white: '#ffffff',
     orange: '#ff6b35',
   };
   ```
4. **每个动画值必须有 `extrapolateLeft: 'clamp'` 和 `extrapolateRight: 'clamp'`**
5. **stagger 动画的延迟必须基于帧数计算**，不得使用毫秒
6. **组件必须是纯函数**，不得使用 `useState`、`useEffect` 等有副作用的 Hook
7. 这份要求是执行代码时的一个标准，不需要输出任何东西给我看

## 💡 设计原则

- **少即是多**：每页只传达一个核心信息，不要塞满内容
- **动画服务内容**：动画是为了强调信息，不是炫技
- **节奏感**：入场不宜太慢，出场不宜太早，展示时间要够读完
- **品牌一致性**：每页都应能看出是同一套视觉体系
- **竖屏思维**：内容集中在屏幕中央 1/3 区域，上下留白是呼吸感
